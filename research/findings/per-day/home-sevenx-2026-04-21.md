⏺ ## 找到原因了 — 是跨机器 git 同步导致的

**触发时间点很干净**：今天 17:16:05 整个家目录里只有一个事件——`~/.claude/plugins/cache/` 的内容被清空，其它什么都没动。没有 shell 历史、没有 cron、没有 systemd timer。这是 **Claude Code 自己做的**。

### 证据链

你的 `~/.claude` 是个 git 同步的 dotfiles 仓库（`git@github.com:SevenX77/cc-rules-global.git`），Mac 和 Linux 共用。关键的 `.gitignore` 配置：

```
plugins/cache/          ← gitignored，每台机器各自重建
plugins/marketplaces/   ← gitignored，同上
```

但是 **`plugins/installed_plugins.json` 是被 git 追踪的**，也就是说插件"登记表"会跨机器同步，但"实际文件"不会。

看一眼追踪的登记表内容：

```json
"superpowers":       "installPath": "/Users/sevenx/.claude/..."  ← Mac 路径
"frontend-design":   "installPath": "/Users/sevenx/.claude/..."  ← Mac 路径
"telegram":          "installPath": "/Users/sevenx/.claude/..."  ← Mac 路径
"claude-hud":        "installPath": "/home/sevenx/.claude/..."   ← Linux 路径（今天才加的）
   version: "unknown"                                            ← 版本标签缺失
```

### 发生了什么

Claude Code 2.1.116 启动时做了一次"插件健康检查"：
1. 读 `installed_plugins.json`，发现 4 个已安装插件
2. 验证每个 `installPath` 是否真实存在
3. 3 个 Mac 路径在 Linux 上必然不存在 → 判定缓存失效
4. **wipe 整个 `plugins/cache/`**（17:16:05 那一刀）
