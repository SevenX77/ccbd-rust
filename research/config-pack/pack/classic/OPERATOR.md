# OPERATOR.md — 经典版拓扑的编排规划

> 通用的任务推进保障体系(三层哨兵 / 停摆分诊 / 高危操作连带 / 证据纪律)见**上级 `../OPERATOR.md`**,那一套与拓扑无关,经典版照用。
> 本文件只补经典版**专属**的编排:席位表、设计→实施→审核→测试闭环、**d1↔r2 调度决策**、operator 边界。

## 一、什么时候用经典版
任务线较集中、要**一条统一审核基准**(单一 r1 席审所有实施代码,而非把审核下沉进多条泳道)+ 一个**独立测试席**压端到端时,用经典版。它比双泳道简单(无泳道层级、无跨泳道交叉审),又比基础版多了发散/执笔分离的设计线和专职测试席。任务能干净拆两条并行线、且要把"实施↔审收"压进泳道最短路径时,才选双泳道。

## 二、席位表(7 席 = master + 6 worker)

| 席位 | 角色 | 推荐 provider 能力 | 关键铁律 |
|---|---|---|---|
| `master` | PM/协调者 | 强语义 + 长上下文 | 不写代码;跨席排期归它;监控锚产物轨不锚 job 状态 |
| `c1` | 实施者(实例1) | 机械精确 codex 类 | 冻结设计逐任务 TDD;代码归 r1 审,**不自审** |
| `c2` | 实施者(实例2) | 同 c1 | c1 的并发副本,各领不同任务,内容对称 |
| `o1` | 设计辩论席 | 发散型,与 master/d1 异族 | **只辩论不执笔**;不落冻结稿、不碰代码 |
| `d1` | 设计主笔 **+ r2 后备审核** | 强语义 claude 类 | **唯一执笔者**;副角色 r2 见下节 |
| `r1` | 审核席 | 强语义 claude 类 | **只审不写**;审核第一责任席,回滚自检必跑 |
| `test`| 测试席 | claude 类(真 e2e)或 codex 类(验收执笔) | 独立测试角色;写/跑验收与 e2e |

## 三、设计→实施→审核→测试闭环
1. **设计**:课题下达 → o1 发散/红队(只辩论)+ master 带事实 → **d1 执笔收敛** → master 与 d1 辩论到冻结。执笔权只在 d1,o1 永不落收敛稿。
2. **实施**:冻结稿交 c1/c2(git-active 任务各自专属 worktree+分支,共享 git 树不能同时 commit),全链路自写自测 TDD。
3. **审核**:实施分支收口后 master 派 r1 审(`git diff <base>..HEAD` + 回滚自检);REJECT 退回实施者修。
4. **测试**:test 席压端到端/验收(安全关键或实施者是发散型时,test 先写 RED 供实施者变绿)。
5. **收口**:全 ACCEPT + CI 绿 → operator push/PR/合并。

## 四、d1↔r2 调度决策(经典版专属编排)
`d1` 是**双角色席**:主角色设计主笔,副角色 r2 后备审核。它戴哪顶帽子**只由 master 派单类型决定**,d1 自己不切换、不自荐审核活。作为 operator,你要向 master 讲清并核对这条协议:

- **默认**:d1 只做设计,**所有代码审核走 r1**。r2 是应急分担,不是常态。
- **何时启用 r2**:**当且仅当**(a) r1 审核饱和——有实施产出排队待审、r1 在途;**且**(b) d1 此刻空闲(手头无设计单)。两条件同时满足,master 才把一份审核单**显式标注"审核任务(r2 模式)"**派给 d1。
- **派什么给 r2(自证闭环回避,最关键)**:master 派给 d1-as-r2 的必须是**与 d1 自己设计裁决无关的独立单元**。凡实现直接源于 d1 冻结的设计、且争议点正是 d1 当初拍板的裁决——那条**留给 r1**,绝不给 d1(否则 = d1 审 d1,自证)。d1 若发现手上 r2 单其实触碰了自己的设计裁决,应 STOP 回退请求改派 r1。
- **退场**:r2 单收工后 d1 回设计主笔本职;r1 始终是审核第一责任席。
- **operator 核对点**:观察日志里记每次 r2 启用(触发时刻/派了哪单/是否触碰 d1 裁决被回退),作为"审核带宽是否够、要不要加独立审核席"的决策弹药。

## 五、operator 的边界(与全栈一致)
- **发版/PR/push 归 operator**:worker 只 commit 不 push;分支一轮收口后 operator push 触发 CI,全绿后 merge(无分支保护的仓库绝不用 `--auto`,挂 CI 看门)。PR 开/合、main、发版同步归 operator。
- **凭据不进全局 `[env]`**:token/密钥走 per-worker 凭据机制,绝不塞进 `ah.toml` 的 `[env]`(会随 home 注入进每个沙箱,含发散型 provider 沙箱)。
- **换血成对换**(CLI + daemon 同一构建),每个大模块合入后换一次,换完那段 dogfood 就是疗效观察窗。
- operator 只动脑不动手,唯一亲手维护的是控制回路本身(三层哨兵,见上级 `../OPERATOR.md`)。
