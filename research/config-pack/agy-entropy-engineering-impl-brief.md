## 已凿定的数学地基(这是前提,不要求你重新论证它对不对;要求你在它之上找工程实现)

1. **语言和代码本质是同一种东西**——都是带不确定性的信息。所以我们要一把跨所有尺度通用的尺子(词 / spec 行 / 函数 / 模块 / 系统),量的是 ΔH:不确定性的变化。
2. **H 与 ΔH 的定义(已锁)**:一个需求落进工程语境 = 若干**决策**;每个决策在当前上下文下有一组**符合最佳实践的候选选项**(=字母表);这些选项上有一个**条件分布**。H = Σ −p·log p。ΔH = 补进一条信息(更多上下文 / 定了另一个决策 / 发现一条约束)之后,这个分布收窄了多少。完美 spec → 每个决策塌成唯一选项 → H = 0。
3. **分布的来源(已锁)**:这个条件分布 = **模型本身对"该选哪个选项"的条件概率**(模型灌满了最佳实践)。H 直接从它算,不需要任何外部"证伪 / 独立见证"机器去测它。
4. **"自信地错"的刻画(已锁)**:模型分布很尖(H 低)但偏离独立最佳实践参考 = 对参考的 KL 散度大。低 H + 高 KL = 自信地错。

6. **架构本原(已收敛到的最简形态)**:一个递归的**语义滤波器**——高熵输入 I → [一个 agent 反复产出 O′ 直到 O′ 满足约束 C] → 在 C 覆盖的那个维度上塌成低熵的 O。C 分两种:**机械断言器**(编译器 / linter / 类型 / 测试:约 70%,便宜、零噪、确定)和 **judge 探针**(语义熵 / 期望信息增益 / critic 模型:约 30% 不可机械化的架构判断,贵且有噪)。递归组合成 DAG,任何尺度同构。**自沉淀** = 某处反复摩擦(worker 重试多次才过 / 合并后 downstream 亮红灯)就把引起摩擦的软规则**自动硬化成一条物理约束**(新 lint 规则 / hook / 类型),注入下一次,单调降未来熵。
