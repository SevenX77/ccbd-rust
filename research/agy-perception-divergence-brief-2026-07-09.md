## 背景(只给问题,刻意不给任何倾向性方案)
ah(Agent Hypervisor,本仓库)是一个 Rust daemon,编排多个跑在 tmux pane 里的交互式 CLI coding agent(claude/codex/antigravity CLI),每个 agent 进程在独立 systemd transient scope(cgroup)里。它需要持续判断每个 agent 的状态:在干活/空闲/任务完成/卡死/在等交互输入。

可用的原始信号(事实清单,无优先级预设):
- OS 进程信号:pidfd 退出通知、exit code、cgroup 成员进程
- agent CLI 的生命周期 hook 回调(如 stop 事件,agent 主动 HTTP/socket 推给 daemon;配置可能丢失,投递可能失败)
- agent CLI 落盘的 transcript/日志文件(可尾随读,格式因 provider 而异)
- tmux pane 的屏幕文本(可 capture,含提示符/spinner/对话框等 UI 痕迹)

历史事实:该系统曾因多个独立监视器基于不同信号各自写状态、先到先得,产生大量误判事故(把回合结束当任务完成、把屏幕幽灵文本当交互对话框、把长任务当卡死)。
