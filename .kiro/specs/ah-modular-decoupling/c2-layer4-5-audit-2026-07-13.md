- Status: implementation audit, current source as of 2026-07-13
- Scope: rewrite of Layer 4 and Layer 5 from `o1-module-map-draft-2026-07-12.md`
- Method: verified paths with `rg --files` / `wc -l`; verified entry symbols with `rg` over `pub` and `pub(crate)` items; checked `src/bin/ah.rs`, `src/bin/ahd.rs`, `src/cli`, `src/rpc`, `src/orchestrator`, `src/monitor`, and `src/provider` references for process ownership.

Process-axis rule used below:

- `ah CLI`: production code runs in `src/bin/ah.rs` or CLI helper call chains.
- `ahd daemon`: production code runs behind `src/bin/ahd.rs`, RPC handlers, orchestrator, monitor, provider tasks, or daemon startup reconciliation.
- `both`: production code is used by both processes.

Observed ownership anchors:

- `src/bin/ahd.rs` imports `db`, `env`, `outbox`, `rpc`, `orchestrator`; calls `env::resolve_state_dir`, `db::init`, `db::system::reconcile_startup_with_tmux_socket_and_gateway`, and `outbox::cold_scan_all_agents`.
- `src/bin/ah.rs` does not initialize or query `db`; it writes hook outbox files with `ah::outbox::*`, builds inactive runtime snapshots with `ah::runtime_events::*`, and reaches live state through RPC.
- `src/cli/rpc_client.rs` uses `state_layout`; `src/cli/setup.rs` also uses neutral state layout.

## Corrections Against Seed Draft

- `db` is not one module for indexing purposes; it is at least 24 source files plus `db/perception/*`, including several 1000-3000+ line ownership centers.
- `runtime_events` does not expose `write_state_snapshot`; verified public entries are snapshot structs, `inactive_runtime_snapshot`, `build_runtime_snapshot`, and `runtime_snapshot_fingerprint`.
- `pane_diff` currently has only `src/pane_diff/mod.rs`; there is no `src/pane_diff/watcher.rs`.
- `completion` has no public `LOG_MONITORS` symbol; the exposed registry API is `register`, `contains`, `cursor_snapshot`, `entry_snapshot`, `update_state`, `take`, `cancel`.
- `prompt_handler` production logic is daemon-side despite the CLI having a `prompt resolve` command; the CLI command is an RPC wrapper.
- `outbox` is explicitly both-process: CLI journal-first write path and daemon cold-scan/consume path.
