# Layer 1-3 Module Audit (c1, 2026-07-13)

Scope: audited current source for Layer 1 (`cli`, `rpc`), Layer 2 (`orchestrator`, `monitor`, `master_revival`, `master_cutover`), and Layer 3 (`tmux`, `sandbox`, `platform`, `systemd_unit`, `agent_io`). Seed draft `.kiro/specs/ah-modular-decoupling/o1-module-map-draft-2026-07-12.md` was used only as a starting point; paths and symbols below were rechecked against the current tree.

Binary ownership anchors:
- `src/bin/ah.rs` (2131 lines): `#[tokio::main] async fn main()` parses the `ah` CLI and directly imports `ah::cli::*`, `ah::tmux::*`, and `ah::systemd_unit::detect_current_scope_or_service`.
- `src/bin/ahd.rs` (466 lines): `#[tokio::main] async fn main()` initializes sandbox/db/tmux/rpc context, calls `monitor::master_watch::rearm_active_master_watches_on_startup`, `orchestrator::spawn_orchestrator_task`, and `rpc::run_server`.
- Cargo boundary caveat: `src/lib.rs` exports all audited modules into the shared `ah` library crate, so both bins compile the same library surface. The ownership axis below reports runtime/entry responsibility, not physical crate isolation.

### `master_cutover`

- Name: `master_cutover`
- One-line responsibility: shared helpers plus CLI/daemon orchestration surfaces for replacing an unmanaged/current master with an ah-managed master and transferring handoff context.
- Source paths and size checked:
  - `src/master_cutover.rs` (296 lines)
  - `src/cli/master_cutover.rs` (393 lines)

- Ownership axis: both processes, split by file. `src/cli/master_cutover.rs` is `ah` CLI code that builds the `session.master_cutover` request and includes `providers.claude.shared_credentials_dir`. `src/master_cutover.rs` is shared helper code used by daemon-side `rpc::handlers::sessions` to write the handoff bundle and seed Claude project conversations. The actual cutover state machine and spawn/ack flow run inside `ahd` via RPC handlers.
